HVAR–1: No Advance Width Mapping

The font for this test case contains a Horizontal Metrics Variation Table whose offsetToAdvanceWidthMapping field is zero. The font, which was kindly contributed by Adobe Systems, had triggered a bug in one version of the FreeType library; see FreeType bug 50170. If your implementation is correct, the advance width should grow as the weight increases. The letters A and B should never overlap.
